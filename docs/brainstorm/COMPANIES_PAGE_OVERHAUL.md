# Companies Page Overhaul Brainstorm

**Theme: Company Management Hub - Command Center for Client Relationships**  
**Goal: Allow superadmins to efficiently manage companies, their users/admins, and see company health at a glance**  
**Anti-Pattern: Avoid buried actions - everything important should be 1-2 clicks away**

> **Created**: December 2, 2025  
> **Status**: âœ… Direction Chosen - Hybrid D+C Approach

---

## Chosen Approach: Hybrid Dashboard + Detail Page

**Decision:** Combine Option D (Dashboard Cards with Quick Actions) + Option C (Full Tabbed Detail Page)

**Flow:**

1. **Companies Dashboard** - Grid of company cards with health indicators and quick stats
2. **Quick Actions Drawer** - Slide-out panel for simple user/admin management
3. **Full Detail Page** - Click company name or "View Details" for full tabbed management interface

This gives us the best of both worlds:

- **At-a-glance overview** of all companies on the dashboard
- **Quick actions** via drawer without leaving the page
- **Full management capabilities** on the detail page when needed

---

## Current State Analysis

### What We Have Now

- Basic company list/grid view
- Click-through to see company tickets
- Separate modals for managing users and admins
- Basic CRUD operations (Create, Read, Update, Delete)
- Ticket views (Table, Kanban, Deadline, Archive)

### Current Pain Points

1. **Scattered Information**: Have to click into each company to see what's happening
2. **Hidden Actions**: User/admin management buried in dropdown menus
3. **No Health Indicators**: Can't quickly see which companies need attention
4. **No Activity Timeline**: No sense of what's been happening with a company
5. **Limited User Management**: Creating users is clunky, no bulk operations
6. **No Company Insights**: No SLA tracking, response times, satisfaction data
7. **No Quick Actions**: Every action requires multiple clicks

---

## Design Goals

1. **At-a-Glance Health**: See company status without clicking
2. **Quick User Management**: Add/remove users without leaving the page
3. **Activity Awareness**: Know what's happening with each company
4. **Actionable Insights**: Surface problems before they escalate
5. **Efficient Workflows**: Reduce clicks for common tasks

---

## API Requirements

### Existing Endpoints (to leverage)

- `GET /companies/` - List companies
- `GET /companies/{id}/` - Company detail
- `POST /companies/{id}/assign_admin/` - Add admin
- `POST /companies/{id}/remove_admin/` - Remove admin
- `POST /companies/{id}/create_user/` - Create user
- `POST /companies/{id}/remove_user/` - Remove user

### New Endpoints Needed

- `GET /companies/{id}/stats/` - Quick stats (open, urgent, overdue counts)
- `GET /companies/{id}/activity/` - Activity feed with pagination
- `POST /companies/{id}/users/bulk/` - Bulk user import
- `POST /companies/{id}/users/{user_id}/reset-password/` - Reset user password

### Stats Endpoint Response Example

```json
{
  "open_tickets": 12,
  "urgent_tickets": 3,
  "overdue_tickets": 2,
  "resolved_this_month": 45,
  "user_count": 5,
  "admin_count": 2,
  "health_status": "critical", // critical | attention | healthy | inactive
  "last_activity": "2025-12-02T14:30:00Z"
}
```
